Product Logic — Stage first-use agent-2 4 3

Conflit de philosophie : la nouvelle nav expose les 12 Stages cliquables, mais le paradigme cockpit est 'l'agent ouvre les Stages avec contexte'. Résultat : la plupart des Stages ouverts manuellement sont vides et renvoient vers l'agent. SimulationStage est le seul gold standard (form-first, self-serve, confirm destructif). VoiceStage = empty state honnête avec fallback. Recommandation : soit doter chaque empty state d'un CTA self-serve, soit assumer le paradigme agent et indiquer 'ouvre depuis le chat'.
