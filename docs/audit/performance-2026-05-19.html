Audit performance — Hearst OS

2026-05-19 · Next.js 15 + React 19 · 3 agents Explore (Sonnet) en parallèle · read-only strict
55 findings P0 · 4 P1 · 27 P2 · 24

Verdict. Le bundle initial est sain grâce aux dynamic() sur les stages et Spline, mais 4 fuites P0 méritent d'être traitées : 2 polls/intervals sans cleanup robuste, 2 requêtes Supabase non bornées (kg_nodes/kg_edges, users), et un risque three.js chargé sync sur OrbitalRing.

Quick wins. 3 actions sous 30 min peuvent réduire ~640–1050 KB du bundle initial et économiser 200–400 ms de LCP mobile : optimizePackageImports, migration next/font/local, et suppression du fetch ThemeHydrator redondant.

Dette structurelle. app/(user)/layout.tsx en "use client" neutralise les Server Components sur toute l'arbo user — chantier de refonte à planifier hors quick wins.

Quick wins (≤ 30 min chacun)

QW1 — optimizePackageImports dans next.config.ts
Ajouter experimental.optimizePackageImports: ['framer-motion', 'zustand', '@supabase/supabase-js', 'zod', 'lucide-react'].
Gain : 80–150 KB gzipped · Effort : 5 min · Severity : P1
QW2 — import "server-only" sur connectors lourds
Ajouter import "server-only"; en première ligne de lib/connectors/google/*.ts, lib/daily-brief/pdf.ts, lib/reports/export/{xlsx,pdf}.ts, lib/engine/runtime/assets/generators/*.ts.
Gain : 500–800 KB si leak évité (googleapis seul) · Effort : 15 min · Severity : P1
QW3 — Migrer la font Satoshi vers next/font/local
Remplacer le <link rel="stylesheet"> externe vers fontshare.com par next/font/local (.woff2 self-hosted) — preload auto + font-display:swap.
Gain : −200 à −400 ms LCP mobile · Effort : 30 min · Severity : P1
QW4 — Supprimer le fetch ThemeHydrator redondant
Le thème est déjà injecté via cookie + data-theme sur <html> dans RootLayout. Le fetch('/api/user/theme', { cache: 'no-store' }) au mount est gratuit.
Gain : −1 requête réseau / mount (~50 ms RTT) · Effort : 10 min · Severity : P2
QW5 — Borner les Supabase kg_nodes / kg_edges
Ajouter .limit(500) + sélectionner colonnes explicites dans lib/memory/kg.ts:515-516 pour éviter OOM sur KG denses.
Gain : −80 % transfert réseau sur KG dense · Effort : 5 min · Severity : P0
