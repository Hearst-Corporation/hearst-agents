Audit comme un utilisateur premier-venu navigant au clavier. Patterns sains déjà adoptés : ConfirmModal sur 6 destructifs, système toast global (24 appels), useModalA11y sur toutes les modales sauf OnboardingTour, lang="fr" root. Les angles morts : le cockpit-shell package n'utilise pas le DS (window.confirm, catch silencieux), les Server Components admin n'ont pas de try/catch sur Promise.all, et plusieurs pages d'écran affichent des CTA non câblés (Marketplace / Apps).

Routes auditées : app/(user)/ · app/admin/ · packages/cockpit-shell/ 2 window.confirm détectés 0 skip-link racine lang="fr" OK · useModalA11y 95% couverture

Top 5 flows à risque

  1. 1.
    app/(user)/components/chat/useRightPanelData.ts — cœur du chat : SSE onerror silencieux bloque le panneau droit en loading infini après coupure réseau, et un .catch(() => {}) sur le fetch asset_generated masque les échecs. Impact : sur 100% des sessions, une coupure ou un 401 fige l'UI sans recours visible. P0 ×2.
  2. 2.
    packages/cockpit-shell/src/chat/ChatHistory.tsx:58,68 — window.confirm() sur deleteChat() et clearAll(), plus catch vide sur l'API DELETE. Incohérence DS, suppression silencieuse côté serveur en cas d'erreur. P0 sur action destructive cumulative (toutes les conversations).
  3. 3.
    app/admin/orchestrator/overview/page.tsx & orchestrator/agents/page.tsx — Promise.all sur fonctions HOM sans try/catch (loadAllContracts, loadCC, loadQuarantine throw explicitement si un fichier manque). L'error.tsx du layout admin capture mais affiche un fallback opaque — l'opérateur ne sait pas quel contrat est manquant. P0.
  4. 4.
    app/(user)/marketplace/page.tsx & apps/page.tsx — boutons "Installer" / "Connecter" sans onClick ni état loading, UI entièrement statique. Le primo-visiteur clique, rien ne se passe. Impression de bug. P1 ×2 sur les premières pages de l'expérience.
  5. 5.
    app/admin/_canvas/RunWaterfall.tsx:70-80 — slider clavier incomplet : role="slider" ne gère que Arrow Left/Right, pas Home/End/Enter/Space. Bornes 0/100% inaccessibles au clavier sur l'outil de debug runs. P0 a11y.

Patterns sains déjà adoptés

  • ConfirmModal appliqué sur 6 destructifs (SimulationStage reset, MissionStepGraph skip, AppDrawer disconnect, WebhooksSection delete, ReportEditor reset/load, NewAgent cancel).
  • Système toast global cohérent (24 appels), hook useToast uniforme.
  • useModalA11y sur ConfirmModal, AssetCompareModal, Commandeur, DocumentParseModal, EnrichmentPreviewModal, VideoQuickLaunchPanel, PopoverShell — focus trap + Escape + restore focus. Seule modale non couverte : OnboardingTour.
  • error.tsx présents sur app/admin/ et app/(user)/ (couvre les routes enfants).
  • <html lang="fr"> dans root layout. Aucun @ts-nocheck dans le scope audité.
  • Pattern :focus-visible systématique dans app/globals.css (7 outline:none, tous accompagnés d'un fallback distinct).

Angles morts récurrents

1. Cockpit-shell hors du DS

Le package packages/cockpit-shell/ n'utilise ni ConfirmModal ni toast ni useModalA11y. window.confirm ×2, :focus-visible absent sur .ct-chat-input. Gap le plus structurant — chaque ajout de feature dans le shell rejoue le pattern d'origine.

2. Server Components admin sans guard

Promise.all sur fonctions HOM qui peuvent throw — masqué par error.tsx du layout, mais le fallback est opaque. Pas de discrimination entre "fichier manquant" et "API down" pour l'opérateur.

3. Pages d'écran avec CTA non câblés

marketplace, apps, archive, notifications affichent des boutons sans onClick ou des filtres morts. Le run OMNISCAN précédent a désactivé les UI mortes sur archive/notifications/marketplace — il reste apps/page.tsx et les filtres reports.

4. Loading.tsx manquants côté admin

5 routes admin/ sans skeleton segment (runs, agents, health, metrics, analytics). Page blanche pendant le SSR/hydration. Réutiliser RowSkeleton/CardSkeleton existants.

Quick wins (ordre suggéré)

  1. Réparer useRightPanelData.ts (P0×2) — set loading=false dans es.onerror + état erreur visible. Effet sur 100% des sessions chat.
  2. Cockpit-shell : adopter le DS — remplacer window.confirm par ConfirmModal, ajouter :focus-visible sur .ct-chat-input. 1 PR ciblée.
  3. Try/catch sur les 3 Server Components admin/orchestrator/* + ajouter app/admin/orchestrator/error.tsx dédié.
  4. OnboardingTour : wrapper avec useModalA11y + autoFocus sur CTA principal.
  5. 5 loading.tsx admin (runs, agents, health, metrics, analytics) — copie-colle de skeleton existants, 10 min chacun.
  6. Compléter RunWaterfall slider (Home/End/Page) — débloque clavier-only sur le debug runs.
  7. Skip-link racine + :focus-visible sur cockpit-shell — gain a11y systémique, 1 commit.
